# Lifolio QA 체크리스트

페이지별로 **정상 동작 / 예외 상황 / 엣지 케이스**를 정리했습니다. 생성형 AI가 실제로 붙는 부분(상담 구조화, 회고 코멘트 요약, DNA 인사이트 등)은 현재 스텁(stub) 로직으로 동작 중이라, 관련 항목은 "지금 붙어있는 로직 기준"으로 표시해뒀고 실제 AI 연동 후 재검증이 필요합니다.

## 테스트 준비사항 (선행 조건)

- [ ] 일반 사용자 계정 1개, 관리자 계정 1개(`ADMIN_EMAILS` 또는 기본 관리자 이메일) 확보
- [ ] 다음 상태의 의사결정을 각각 최소 1개씩 준비: `IN_PROGRESS`(상담 중), `DECIDED`(회고 대기), `COMPLETED`(회고 1건), `COMPLETED`(회고 2건 이상 — 다시 회고하기 검증용)
- [ ] 회고 3건 이상 완료된 계정 1개(리포트 Level 1 잠금 해제 검증용)
- [ ] 라이브러리에 **승인된** 본인 케이스 1개(추가 회고 업데이트 플로우 검증용)
- [ ] 관리자 승인 대기 중인 신규 제출 1건 + 추가 회고 업데이트 1건

## 7. AI 상담 (`/consult`)

> AI 구조화/응답 로직은 현재 스텁 상태 — 실제 AI 연동 후 재검증 필요.

**정상 동작**
- [ ] 신규 상담 시작 시 intro 단계 + 추천 주제(quickTopics) 노출
- [ ] 고민 입력 후 필요한 정보(배경/상황/옵션/기준/우려)를 대화형으로 순차 수집
- [ ] 필요한 정보가 모두 채워지면 자동으로 옵션 입력 폼 → 옵션별 기대/걱정 입력 폼 → 판단 기준 입력 폼으로 전환
- [ ] 모든 단계 완료 시 요약(SummaryCard) 노출
- [ ] 요약 화면에서 최종 선택 + 확신도 입력 후 진행
- [ ] 회고 예정일 선택(달력 UI) 또는 "나중에 정할게요" 선택 가능
- [ ] 완료 시 `Decision.status`가 `DECIDED`로 전환되고, 홈의 "회고 대기 중"에 정상 노출
- [ ] `?decisionId=`로 기존 `IN_PROGRESS` 상담 재개 시 기존 입력값 복원

**예외 상황**
- [ ] 필수 필드 없이 다음 단계 진행 시도 시 막히는지
- [ ] 상담 중 새로고침 시 진행 상황 유지 여부(또는 처음부터 재시작 — 의도 확인)
- [ ] 존재하지 않거나 타 사용자 소유의 `decisionId`로 재개 시도 시 안전 처리
- [ ] 이미 `DECIDED`/`COMPLETED`인 `decisionId`로 접근 시도 시 처리

**엣지 케이스**
- [ ] 옵션 2개 미만으로 진행 시도
- [ ] 판단 기준/우려 항목 0개로 진행 시도
- [ ] 매우 긴 텍스트(수백~수천자) 입력
- [ ] 옵션 제목 중복 입력
- [ ] 확신도(confidence) 0/100 경계값
- [ ] 회고 예정일을 과거 날짜로 설정(즉시 회고 대기로 분류되는지)
- [ ] 상담 미완료 상태로 오래 방치 후 재개

## 8. 회고 작성/수정 (`/capsule`)

**정상 동작 — 신규 작성**
- [ ] `decisionId` 없이 접근 시 "회고할 결정이 없어요" 빈 상태
- [ ] 회고 대기 중 `decisionId`로 접근 시 결정 제목/최종 선택/경과일 정상 표시
- [ ] 필수 필드(실제 결과 / 선택하고 살아보니 무엇이 달랐나요 / 다시 선택할지 / 왜 그런 선택을 하셨나요) 모두 입력해야 제출 버튼 활성화
- [ ] "다른 사람을 위한 한마디"는 선택 입력(비워도 제출 가능)
- [ ] 만족도 슬라이더 0~100 정상 동작(기본값 85)
- [ ] 제출 성공 시 `/home` 이동 + 저장 알림 노출
- [ ] 첫 회고 제출 시 `Decision.status`가 `COMPLETED`로 전환되고 관리자 승인 큐에 자동 등록

**정상 동작 — 수정 모드 (`?reflectionId=`)**

**예외 상황**
- [ ] 텍스트 필드 글자수 제한(각 500자, 한마디는 200자) 초과 입력 제한
- [ ] 제출 중 네트워크 오류 시 에러 메시지 노출 + 재시도 가능
- [ ] 더블클릭 등으로 중복 제출 시도 시 중복 레코드 생성 방지 여부
- [ ] `IN_PROGRESS`(미결정) `decisionId`로 접근 시 서버 409 및 적절한 처리
- [ ] 존재하지 않거나 타 사용자 소유의 `decisionId`/`reflectionId` 접근 시 안전 처리

**엣지 케이스**
- [ ] 만족도 0/100 경계값 제출
- [ ] 텍스트 필드에 이모지·특수문자·여러 줄바꿈 입력

## 13. 리포트 (`/report`)

**정상 동작**
- [ ] 데이터 없는 신규 사용자는 안내 문구 + 잠금 탭 노출
- [ ] Level 0(결정 성향 프로필): 결정 1건이라도 `DECIDED`되면 잠금 해제
- [ ] Level 1(AI 인사이트): 회고 3건 이상 완료 시 잠금 해제, 미만 시 진행률(N/3) 안내
- [ ] Level 2~5: 항상 잠금 상태 + 안내 문구(MVP 범위 밖)
- [ ] 카테고리별 통계, 판단 기준 빈도, 평균 소요시간, 다시 선택 비율, 평균 만족도 정상 계산
- [ ] 잘된 결정/아쉬운 결정 목록 정상 분류·노출
- [ ] 만족도 추이 그래프 정상 렌더링

**예외 상황**
- [ ] 회고 정확히 2건일 때 Level 1이 여전히 잠김, 3건째부터 해제되는 경계값 확인
- [ ] DNA 스냅샷 생성 실패 시 페이지 전체가 깨지지 않는지

**엣지 케이스**
- [ ] 만족도가 전부 동일한 값일 때 "잘된/아쉬운 결정" 분류 경계(70 이상/50 미만) 동작
- [ ] "다시 선택 안 함"인데 만족도는 높은 모순적 데이터의 분류 확인(만족도<50 OR 다시선택=NO 조건)
- [ ] 새 회고 추가 직후 재방문 시 스냅샷이 최신 회고 수 기준으로 재생성되는지
- [ ] 판단 기준 종류가 매우 많을 때 상위 7개만 노출되는지

## 14. 관리자 — 라이브러리 검토 (`/admin/library-review`)

**정상 동작**
- [ ] 관리자만 접근 가능, 비관리자는 404
- [ ] 신규 케이스 승인 대기 목록과 "추가 회고 업데이트" 목록이 구분되어 노출
- [ ] 신규 케이스 카드에서 모든 필드(제목/카테고리/태그/만족도/다시 선택 여부 등) 수정 가능
- [ ] "승인하고 라이브러리에 올리기" 클릭 시 실제 `/library`에 게시되고 큐에서 제거
- [ ] "거부" 클릭 시 큐에서 제거되고 재노출 안 됨
- [ ] 추가 회고 업데이트 카드에서 내용 수정 후 승인 시 해당 공개 케이스의 "그 이후 이야기"에 반영
- [ ] 승인/거부 처리 후 헤더 대기 건수 배지 즉시 갱신

**예외 상황**
- [ ] 승인/거부 API 실패 시 목록에서 제거되지 않고 에러 메시지 노출(최근 수정 검증됨)
- [ ] 만족도 필드에 숫자가 아닌 값 입력 후 승인 시도 시 처리
- [ ] 같은 항목을 다른 탭/관리자가 먼저 처리한 뒤 재처리 시도(경쟁 상태) 시 적절한 에러
- [ ] 아직 라이브러리 미승인(`PENDING`/`REJECTED`) 결정의 추가 회고가 큐에 잘못 등록되지 않는지(승인된 것만 큐잉)

**엣지 케이스**
- [ ] 승인 대기 0건일 때 안내 문구
- [ ] 신규 케이스 + 추가 업데이트가 동시에 여러 건 있을 때 두 섹션 모두 정상 노출
- [ ] 필수 텍스트 필드를 비운 채 승인 시도
- [ ] 매우 긴 텍스트 입력 후 승인 시 공개 페이지 레이아웃
- [ ] 같은 결정에 "다시 회고하기"를 여러 번 해서 추가 업데이트가 여러 건 쌓였을 때 각각 독립적으로 승인/거부되고, 승인 순서대로 "그 이후 이야기"에 누적되는지
